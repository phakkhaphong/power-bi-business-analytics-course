## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

### 1. Do Not Disclose Publicly

- **Do not** create a public GitHub issue
- **Do not** discuss the vulnerability in public forums
- **Do not** share the vulnerability with others until it's been addressed

### 2. Report Privately

Send an email to: [INSERT SECURITY EMAIL ADDRESS]

Include the following information:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes
- Your contact information

### 3. What to Expect

- **Acknowledgment**: We will acknowledge receipt within 48 hours
- **Investigation**: We will investigate the issue promptly
- **Updates**: We will provide regular updates on our progress
- **Resolution**: We will work to resolve the issue as quickly as possible

### 4. Disclosure Timeline

- **Initial Response**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: Within 30 days (or as soon as possible)
- **Public Disclosure**: After the issue is resolved and patches are available

## Security Best Practices

### For Users

- **Keep Software Updated**: Always use the latest version of Power BI Desktop
- **Use Strong Passwords**: Use strong, unique passwords for all accounts
- **Enable Two-Factor Authentication**: Enable 2FA where available
- **Be Cautious with Data**: Don't share sensitive data unnecessarily
- **Regular Backups**: Keep regular backups of your work

### For Contributors

- **Secure Development**: Follow secure coding practices
- **Code Review**: All code changes must be reviewed
- **Dependency Management**: Keep dependencies updated
- **Access Control**: Limit access to sensitive information
- **Documentation**: Document security considerations

### For Instructors

- **Data Privacy**: Protect student data and privacy
- **Secure Communication**: Use secure communication channels
- **Access Control**: Control access to course materials
- **Regular Updates**: Keep course materials updated
- **Incident Response**: Have a plan for security incidents

## Data Protection

### Personal Data

We are committed to protecting personal data in accordance with applicable privacy laws:

- **Collection**: We collect only necessary data
- **Processing**: We process data lawfully and transparently
- **Storage**: We store data securely
- **Sharing**: We share data only when necessary and with consent
- **Retention**: We retain data only as long as necessary

### Student Data

- **Confidentiality**: Student data is kept confidential
- **Access Control**: Access is limited to authorized personnel
- **Encryption**: Data is encrypted in transit and at rest
- **Backup**: Regular backups are maintained
- **Disposal**: Data is securely disposed of when no longer needed
